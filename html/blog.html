What is robots.txt and how have you configure it for your site?

Robots.txt is a simple text file in your site root, www.mypage.com/robots.txt. Web site owners use the robots.txt file to give instructions about their website to web robots. It is called The Robots Exclusion Protocol. A robot wants to visit a certain website URL, before it does that it checks the robots.txt file, if that website has one. It finds things like, User-agent, Disallow and allow. User-agent: Means this section applies to all robots. Disallow: Tells the robot which pages or any page at the site. If you write Disallow /html/contact.html you are not allowed to enter that page. Allow: And of course this is which pages your are allowed to visit.


What is humans.txt and how have you configure it for your site?

Humans.txt is a simple text file in your site root. It is a file that contains information about the different people who have contributed to the website. In big projects there is alot of people involved and everybody who humans been apart of the project can be in the list, as long as they want to. You can also add an author tag to the head tag of the site.


What is Open Graph and how do you make use of it?

When sharing your website on different platforms, open graph defines what or which elements are displayed. Four required properties for every page are: og:title, og:type, og:image, og:url. og:title, the title of your object as it should appear within the graph. og:type, the type of your object, for example video.movie or website. og:image, the image url which should represent your object within the graph. og:url, the canonical url of your object that will be used as it's permanent ID in the graph.


Why is it important to seperate the layout and design (css) from content (html)?

Because the code becomes easier to read for other develepors and it is easier to maintain a good structure in your code. With an external css file, the style can be reusable. Smaller projects with a couple html files can have only one css file, but when projects become massive it is maybe better to have a couple of css files, just to mantain good structure. It has an impact on performance, smaller code chunks are loaded faster, and that is also a reason to only have one css file, browsers will cache css files resulting in quicker loading.
